# Admin Role

## Purpose

이 문서는 pickup-service 운영 관리자 관점의 작업 범위와 확인 포인트를 정리한다.
어드민 기능 구현과 운영 흐름 검증 기준으로 사용한다.

## Scope

- `apps/admin`
- 관리자 전용 API 요구사항
- 운영 상태 변경 흐름

---

## Main Responsibilities

- 요청 목록 확인
- 요청 상세 확인
- 상태 변경 처리
- 내부 메모 관리
- 운영 관점 데이터 검증

---

## Key Rules

- 관리자 기능은 일반 사용자와 명확히 분리되어야 한다.
- 상태 변경은 허용된 전이만 가능해야 한다.
- 내부 메모는 절대 사용자에게 노출되면 안 된다.
- 목록 화면은 처리 효율과 정보 밀도를 우선한다.

---

## Main Deliverables

- 관리자 로그인
- 요청 목록
- 요청 상세
- 상태 변경 액션
- 내부 메모 UI

---

## Collaboration

- Backend와 상태 전이 규칙을 맞춘다.
- Frontend와 운영자 관점 정보 우선순위를 맞춘다.
- Planner와 운영 정책 변경 여부를 확인한다.
